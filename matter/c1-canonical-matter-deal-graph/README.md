# C1 — Canonical matter/deal graph and world-state engine (the hidden oracle)

Creates the hidden, typed oracle from which synthetic facts, documents, issues, calculations, and expected work products are derived.

Plane: C (Canonical matter and evidence) · Phase: 2 — Canonical-state compiler proof (`BLOCKED`)

## Build

- [ ] Matter identity/lifecycle and buyer/target/seller/counterparty graph
- [ ] Subsidiaries/beneficial-ownership, capitalization/securities, transaction structure/timeline
- [ ] Contracts/amendments/orders and assets/liabilities/debt/liens objects
- [ ] Employees/benefits/IP/data/regulatory/litigation objects, values/dates/currencies
- [ ] Evidence-availability state, known unknowns, closed-world completeness assertions
- [ ] Planted non-issues and intended-ambiguity templates

**Depends on:** A1, B3/B4. Feeds C2–C4, D2/D3. Only C2/G2 may mutate a released graph version.
**Posture:** Build as core differentiated infrastructure; integrate generic graph/schema storage only.
**Done when:** schema validity holds; all referential/temporal/financial invariants pass; every expected conclusion traces to underlying facts + evidence status; exact seed replay works.
**Avoid:** internally impossible deals; absence/evidence conflation; facts used by criteria but absent from schema; naive Cartesian sampling producing unrealistic combinations.
