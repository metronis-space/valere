# C3 — Document, spreadsheet and virtual-data-room renderer

Renders a realistic observable matter packet from canonical state while retaining field-to-span provenance.

Plane: C (Canonical matter and evidence) · Phase: 3 — Document-world proof (`BLOCKED`)

## Build

- [ ] Clause/template renderer, contract/amendment assembler, defined-term/cross-reference engine
- [ ] Disclosure-schedule generator and cap-table/financial/funds-flow spreadsheet renderer
- [ ] Board/stockholder materials, policy/log/incident files, email/Q&A/correspondence, litigation/regulatory records
- [ ] Signatures/notices/consents, VDR folder/index, distractor/document-noise renderer
- [ ] PDF/DOCX/XLSX conversion and a field-to-span rendering-provenance map

**Depends on:** C1/C2, A2/A3, B3. Feeds C4, D2, E1/E2, F1.
**Posture:** Build domain render orchestration + provenance; integrate mature document/spreadsheet/PDF/OCR libraries inside the approved rights boundary.
**Done when:** files open/parse; required fields and issues survive rendering; defined terms/cross-references/signatures/math are valid; provenance covers every material field.
**Avoid:** visually plausible but semantically broken files; stale/superseded amendments visible as current; the renderer and validator sharing the same bug.
