# Phase 0 configuration contracts

- `*.template.yaml` files use the executable v1 schemas and intentionally stay
  `DRAFT`/`TBD` until real owners approve them.
- `*.example.yaml` files preserve the public Harvey LAB fictional scenario as
  research illustrations. They are not production contracts and the compiler
  will not accept them as a Phase 0 exit.

Validate the production templates from the repository root:

```bash
python3 -m scope validate \
  --manifest scope/configs/ScopeManifest.template.yaml \
  --rights scope/configs/RightsRegister.template.yaml \
  --governance scope/configs/DataGovernancePolicy.template.yaml \
  --authority scope/configs/AuthoritySignoffMatrix.template.yaml
```

The expected status is `BLOCKED` until the external decisions listed in
[`scope/README.md`](../README.md) are resolved.

`demo/` is generated by `python3 -m scope demo-generate`. It is a Harvey LAB
M&A-only POV pinned to one `tasks/corporate-ma` change-of-control diligence
task. Its fictional contracts exercise every gate and may produce only
`TEST_READY`. That artifact may feed Phase 1 POV work, but never a real Phase 0
exit, production authorization, or real-client claim.

Phase 1 writes its JSON bundle and test-only exit artifact into the same
directory without adding YAML outside `scope/configs/`:

```bash
python3 -m truth demo-generate --out-dir scope/configs/demo
```

Phase 2 then writes its test-only canonical-state exit artifact alongside them:

```bash
python3 -m matter demo-generate
```

Phase 3 renders and independently validates its document world in the same
ignored demo directory:

```bash
python3 -m matter document-world
```
